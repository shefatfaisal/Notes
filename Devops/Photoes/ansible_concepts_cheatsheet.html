Architecture — how it works
Core concepts

Inventory

  • List of managed hosts (IP or hostname)
  • Groups hosts logically (e.g. webservers, dbs)
  • Static (.ini/.yaml) or dynamic (script)

Playbook

  • YAML file — the "what to do"
  • Contains one or more plays
  • Each play targets hosts and lists tasks

Task

  • A single unit of work (install, copy, start)
  • Calls a module with arguments
  • Runs in order, top to bottom

Module

  • Built-in "verb" (apt, yum, copy, service…)
  • Idempotent — safe to run repeatedly
  • 3,000+ modules in Ansible Galaxy
More concepts

Handler

  • Task triggered only when notified
  • Runs once at end of play (e.g. restart nginx)
  • Prevents redundant restarts

Variable (var)

  • Reusable values across tasks
  • Sources: playbook, inventory, vars_files, CLI
  • Templated with Jinja2: {{ var_name }}

Role

  • Reusable bundle: tasks + vars + handlers + templates
  • Standard folder structure
  • Share via Ansible Galaxy

Fact

  • Auto-collected host info (OS, IP, RAM…)
  • Gathered at play start via setup module
  • Use as variables: {{ ansible_os_family }}
Playbook anatomy — annotated example
--- # playbook.yml - name: Install and start nginx ← play name hosts: webservers ← inventory group become: true ← sudo/root vars: http_port: 80 tasks: - name: Install nginx ansible.builtin.apt: name: nginx state: present ← idempotent notify: Restart nginx ← triggers handler - name: Copy config ansible.builtin.template: src: nginx.conf.j2 dest: /etc/nginx/nginx.conf handlers: - name: Restart nginx ansible.builtin.service: name: nginx state: restarted

Play

  • Maps a group of hosts to a list of tasks
  • Sets become (privilege escalation), vars, etc.

Idempotency

  • state: present = ensure installed, not "install every time"
  • Run the same playbook 10× → same result

Jinja2 templating

  • {{ http_port }} — inject variable
  • {% if %} — conditionals in templates
  • Used in tasks, templates, conditionals

Handler flow

  • Task runs → notify: Restart nginx
  • Handler queued (not run yet)
  • End of play → handler fires once
Inventory (static .ini)
# hosts.ini [webservers] web-01 ansible_host=10.0.0.1 web-02 ansible_host=10.0.0.2 [databases] db-01 ansible_host=10.0.0.10 [production:children] ← group of groups webservers databases [all:vars] ansible_user=ubuntu ansible_python_interpreter=/usr/bin/python3
Role directory structure
roles/ └── nginx/ ├── tasks/ │ └── main.yml ← task list ├── handlers/ │ └── main.yml ← handlers ├── templates/ │ └── nginx.conf.j2 ← Jinja2 templates ├── vars/ │ └── main.yml ← role variables ├── defaults/ │ └── main.yml ← default values (lowest priority) └── meta/ └── main.yml ← role dependencies
Essential modules

Package mgmt

  • apt — Debian/Ubuntu
  • yum / dnf — RHEL
  • pip — Python packages

Files

  • copy — push local file
  • template — Jinja2 render
  • file — perms, dirs, links

System

  • service — start/stop
  • systemd — systemd units
  • cron — schedule jobs

Execution

  • command — run cmd (no shell)
  • shell — run via /bin/sh
  • script — local script on remote

Users & groups

  • user — create/modify users
  • group — manage groups
  • authorized_key — SSH keys

Cloud & network

  • uri — HTTP requests
  • aws_ec2 / gcp_compute
  • docker_container
Key CLI commands

Run a playbook

ansible-playbook playbook.yml -i hosts.ini ansible-playbook playbook.yml --check # dry run ansible-playbook playbook.yml --diff # show changes ansible-playbook playbook.yml --tags install

Ad-hoc commands

ansible webservers -m ping ansible all -m shell -a "df -h" ansible db-01 -m apt -a "name=curl state=present" -b

Galaxy (share roles)

ansible-galaxy install geerlingguy.nginx ansible-galaxy init my_role ansible-galaxy collection install community.docker

Vault (secrets)

ansible-vault encrypt secrets.yml ansible-vault edit secrets.yml ansible-playbook pb.yml --ask-vault-pass
Execution flow
Quick tips & gotchas

Variable precedence (low→high)

  • Role defaults
  • Inventory vars
  • Playbook vars
  • Extra vars (-e) — highest

Tags — run partial plays

  • Tag tasks: tags: install
  • Run only: --tags install
  • Skip: --skip-tags install

Conditionals

  • when: ansible_os_family == "Debian"
  • when: result.rc != 0
  • Supports Jinja2 expressions

Loops

  • loop: [nginx, curl, git]
  • Use {{ item }} inside task
  • Replaces old with_items

Vault (encrypt secrets)

  • Encrypt any var or file
  • Never commit plain passwords
  • Integrates with playbooks cleanly

Idempotency rules

  • Prefer state: over raw shell cmds
  • Use creates: with command module
  • Test with --check first
