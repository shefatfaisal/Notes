concept 01
What is OSPF?
The one-line definition IGP Link-State

OSPF is a routing protocol that runs inside an organization's network. Every router shares information about its own links with every other router in the same area, building a shared map of the topology, then independently runs Dijkstra's algorithm to find the best path to every destination.

Mental model: Every router is a city that posts its own road map on a public bulletin board. Every other city reads every posted map. Each city then independently calculates its own best GPS route to every other city using all the maps combined.
IGP — Interior Gateway Protocol

Runs inside one organization's network (one Autonomous System). Knows the internal topology in full detail.

EGP — Exterior Gateway Protocol

Runs between organizations (BGP). Knows only summaries — not internal topology of other networks.

concept 02
LSDB — Link State Database
What it is per router, per area

Every OSPF router maintains its own LSDB — a collection of Link State Advertisements (LSAs) describing every router and link in the area. All routers in the same area end up with identical copies through flooding.

Area 1 — every router holds identical copy: LSA from R1: "I have these interfaces → 10.1.1.1, 10.1.2.1" LSA from R2: "I have these interfaces → 10.1.1.2, 10.1.3.1" LSA from R3: "I have these interfaces → 10.1.2.2, 10.1.3.2" ↓ fed into Dijkstra's algorithm ↓ Routing table for R1: 10.1.3.0/24 → via R2, cost 2 10.1.2.0/24 → via R3, cost 1
Key insight: The LSDB is not one shared database floating in the cloud. It is a copy stored locally on every single router. Flooding exists to keep all those copies synchronized. If even one router has a different copy, Dijkstra produces different results and routing inconsistencies occur.
concept 03
Areas & The Hierarchy Problem
Flat network — the problem

Every router knows every other router. One link failure triggers Dijkstra recalculation on every router in the entire network. Flooding spreads everywhere. LSDB grows without bound.

Like a village where every pothole gets announced to every person simultaneously — fine at 50 people, catastrophic at 50,000.
Hierarchical (areas) — the fix

Routers only know their own area's full topology. A link failure in Area 1 triggers recalculation only in Area 1. Area 2 and Area 3 are completely unaffected.

Like a city divided into districts — a broken road in downtown only needs to be announced to downtown residents, not the entire country.
The blast radius rule: Dijkstra runs per area. Flooding stops at area borders. A flapping link in Area 3 can never force Area 1 routers to recalculate. This containment is the entire reason areas exist.
concept 04
The Three Unbreakable Area Rules
1
Area 0 (backbone) must always exist Area 0 is the mandatory central hub. There is no OSPF without it. It is the highway that all inter-area traffic must pass through.
2
Every other area must connect directly to Area 0 Area 1 cannot talk to Area 3 directly. Traffic must flow Area 1 → Area 0 → Area 3. This creates a mandatory star topology with Area 0 at the center. This prevents routing loops.
3
Area 0 must be contiguous — no splits All backbone routers must be physically connected to each other. A split backbone means the two halves cannot synchronize their LSDBs — routing collapses. Fix: virtual links (a workaround, not a design goal).
Legal topology (star): Area 1 ──ABR──┐ Area 2 ──ABR──┤── Area 0 (backbone) Area 3 ──ABR──┘ Illegal topology (no direct area-to-area links): Area 1 ── Area 2 ── Area 3 ← Area 2 cannot reach Area 0 here
concept 05
Router Types — Who Does What

Router type is determined entirely by where its interfaces sit, not by any manual configuration.

Internal router

All interfaces in one non-backbone area

The most common router. Only knows its own area. Lightest processing load. Generates Type 1 LSA only.

Employee who only knows their department floor plan.
Backbone router

All interfaces in Area 0 only

Keeps the central highway running. Handles inter-area forwarding. Generates Type 1 LSA for Area 0.

Highway interchange worker — manages the road everyone uses.
ABR — Area Border Router

Interfaces in Area 0 + at least one other area

The most critical router type. Maintains two separate LSDBs. Runs Dijkstra twice. Generates Type 3 Summary LSAs — the only way routes cross area boundaries. It is a translator, not a mixer.

Regional manager who summarizes their region to HQ without leaking internal details.
ASBR — AS Boundary Router

One interface in OSPF AS, one in external AS

The edge of your OSPF world. Connects to BGP, EIGRP, RIP, or static routes. Redistributes external routes into OSPF as Type 5 External LSAs.

Customs officer — everything entering or leaving the country passes through them.
concept 06
LSA Types — The Data Records
LSAs are the books in the library. They live permanently in the LSDB. Each type is a different category of book describing a different aspect of the network.
Type Name Who generates What it describes Scope
1 Router LSA Every router Its own interfaces & directly connected networks Stays in area
2 Network LSA DR only (broadcast segments) All routers attached to that broadcast segment Stays in area
3 Summary LSA ABR only Routes from another area — summarized, not raw topology Injected into Area 0
4 ASBR Summary ABR How to reach an ASBR Crosses areas via ABR
5 External LSA ASBR only Routes from outside the OSPF AS entirely Floods entire AS
Critical: Type 2 and Type 3 are the most confused pair. Type 2 never leaves its area — it describes a broadcast segment internally. Type 3 is what the ABR creates fresh to advertise that area's routes into Area 0. The ABR reads Type 2, then creates a brand new Type 3. The Type 2 itself never crosses the boundary.
What actually crosses an area boundary: Area 1 internal: Type 1 (R1) + Type 1 (R2) + Type 2 (DR) ↓ ABR reads all of these ↓ ABR generates → Type 3 Summary LSA → floods into Area 0 "10.1.0.0/16 is reachable via me, cost 15" The raw Type 1s and Type 2s from Area 1 NEVER enter Area 0.
concept 07
OSPF Message Types — The Delivery System
Message types are the postal system. They are temporary — sent, received, done. They do not live in the LSDB. LSA types are the books. Message types are the delivery trucks that move books around.
MSG 1
Hello Sent every 10 seconds (Ethernet) to multicast 224.0.0.5. Discovers neighbors, negotiates compatibility, and maintains the relationship as a heartbeat. Contains must-match fields: Hello interval, Dead interval, Area ID, Subnet mask, Auth. If any mismatch — no neighbor formed, no error shown, packet silently ignored.
MSG 2
DB Description "Here is a catalog of what's in my LSDB." Sends LSA headers only — no full data. Like a library sending its card catalog, not the actual books.
MSG 3
LS Request "You have these LSAs that I don't. Please send me the full versions." The router compares DB Descriptions and identifies gaps.
MSG 4
LS Update The actual full LSA data. This is what gets stored in the LSDB. The most important message for building topology knowledge.
MSG 5
LS Acknowledgement "I received your LS Update." Just a confirmation using LSA headers — not the full data again. OSPF uses reliable delivery so every LS Update must be acknowledged.
concept 08
Hello Packet — Deep Dive
Must-match fields gatekeeper

Mismatch on any of these = neighbor silently rejected.

  • Hello interval (default 10s Ethernet)
  • Dead interval (default 4× hello = 40s)
  • Area ID
  • Subnet mask
  • Authentication password
Informational fields coordination

Used to coordinate, not gatekeep.

  • Router ID (who sent this Hello)
  • DR / BDR addresses (election info)
  • Router priority (used in DR election)
  • Neighbor list (Router IDs already seen)
The neighbor list — why it's clever

A Hello from R1 doesn't prove R2 can hear R1. The neighbor list solves this. R2 lists R1's Router ID in its own Hello reply. When R1 sees its own ID listed — bidirectional confirmed. This state is called 2-way. Only after 2-way do routers proceed to full LSDB synchronization.

Step 1: R1 → Hello → R2 (neighbor list: empty) Step 2: R2 → Hello → R1 (neighbor list: R1) ← "I see you" Step 3: R1 sees its own ID → 2-way state reached Step 4: Proceed to DB Description exchange (msgs 2-5)
concept 09
Network Types

Two separate classifications — physical type (what hardware) and traffic role (how traffic uses it). They are independent.

Point-to-Point physical type

Exactly two routers on the link (e.g. HDLC, leased line). No DR election needed — only one possible neighbor. Generates Type 1 only.

Broadcast Multi-Access physical type

Many routers on a shared medium (Ethernet switch). DR elected to prevent N×(N-1)/2 adjacency explosion. DR generates Type 2 Network LSA for the segment.

Transit network traffic role

Traffic passes through — source and destination are elsewhere. The network is just a road. Example: link between two backbone routers.

Stub network traffic role

Dead end. All traffic either originates from or is destined to addresses inside this network. Nothing just passes through. Example: office LAN with end-user PCs.

DR election trigger: The DR is elected on broadcast segments — not based on how many routers are in the area. An area with 100 routers on point-to-point links has zero DR elections. A single Ethernet segment with 3 routers triggers one.
concept 10
OSPF Metric — Cost
RFC says: "a single dimensionless metric"

The OSPF RFC deliberately leaves cost undefined — just a number. Each vendor implements it their own way. Cisco uses interface bandwidth.

Cost = Reference BW / Interface BW Cisco default: reference BW = 100 Mbps · cost is always an integer (minimum 1)
LinkSpeedFormulaCost
Serial T11.5 Mbps100 / 1.564 — very expensive
Ethernet 10M10 Mbps100 / 1010
Fast Ethernet100 Mbps100 / 1001 — cheapest
Gigabit1000 Mbps100 / 1000 = 0.11 ← rounds down!
10 Gigabit10,000 Mbps100 / 10000 = 0.011 ← same as GigE!
The modern problem: Cost cannot go below 1 (integer). A 100 Mbps and a 10 Gbps link both get cost 1. OSPF treats them identically. Fix: raise reference bandwidth to 10,000 Mbps so all link speeds produce distinct values.
Path cost = sum of exit interface costs. Incoming interface cost is never counted — only the cost of the interface a packet leaves through. Dijkstra finds the path with the lowest total sum across all hops.
concept 11
Bandwidth — What it Actually Means
Physical definition

Bandwidth is how much data can flow through a wire per second. It is a measure of capacity — the width of the pipe, not the speed of the water inside.

Not a pipe — a highway. 1 Mbps = dirt road (1 car/sec). 100 Mbps = 4-lane highway. 10 Gbps = an airport. All cars travel at the same speed (light). The difference is how many can travel simultaneously.
What bandwidth IS
  • More data in flight simultaneously
  • Larger files finish transferring faster
  • More concurrent connections supported
What bandwidth is NOT
  • Speed of individual packets (always ~light)
  • Lower latency
  • More reliability
What sets the speed: The slowest of three things — the cable category (Cat5e, fiber), the port hardware on each router/switch, and what both sides negotiate during auto-detection. Plug a Cat6 cable into a 100 Mbps port — you get 100 Mbps, not Cat6's potential.
concept 12
Autonomous System & ASN
What an AS is

An Autonomous System is a collection of IP networks under one organization's control that presents a single routing policy to the internet. Each AS gets a globally unique ASN (Autonomous System Number) from IANA.

Country code analogy: +1 = USA, +44 = UK, +91 = India. ASN is the same idea for networks. Inside your country you use local systems (OSPF). To call another country you need the country code (BGP + ASN).
Home network — no AS needed

1 public IP from ISP. 1 ISP connection. No routing control. ISP makes all decisions. You are a passenger.

Organization — needs an AS

Owns thousands of IPs. Connects to multiple ISPs. Needs to control routing policy. Announces its own prefixes. It drives itself.

OSPF vs BGP — where each operates: [Your Company AS 65001] [Internet — AS 7018, AS 15169...] ┌─────────────────────┐ │ OSPF runs here │──ASBR──BGP──→ other ASes │ (inside your AS) │ └─────────────────────┘ OSPF = knows every router inside your building BGP = knows only which ASes advertise which prefixes
the big picture
One Mental Model to Rule Them All
Imagine a country (the AS). Inside it are states (areas). Each state has its own road map (LSDB). State officials (Internal routers) only know their state's roads. The governor's office (ABR) sits on the state border — it knows both the state roads and the national highway (Area 0) and sends short summaries across the border, never the full state road map. The immigration officer (ASBR) handles everything coming in from foreign countries (external ASes). The federal highway (Area 0) must always exist, always be connected, and all states must connect to it — never directly to each other.

Every road has a cost (OSPF cost). Wider roads (higher bandwidth) cost less to use. The GPS (Dijkstra) always routes through the cheapest total path.
