S3 supports **WORM(Write once Read Many)** model

by default versoning is disabled

we can enable versioning in bucket level and apply object level

how many times we upload fille those many versions will be created 
if we delete the orginal version delete marker will be applied to that version

if we remove the delete marker orginial version will be restored but older version wiil not be restored for that we have to download older version and upload

we can suspended the versioning but no version will be created if we delete and remove the delete marker then orginal version will not be restored
