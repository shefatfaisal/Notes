**RDS(Relational Database Services)**

**RDS** is a service where we can setup, configure, maintain RDBMS databases

RDS DB instances, RDS is a database service

RDS is not a database

this service supports 7 engines
- Mysql
- oracle
- Mssql
- postgresql
- mairadb
- aurora - AWS
- ibm db2

this service creates  **DB Instance**
Client only access this instance through **endpoint**


If we want to do database migration we can use **DMS(Database Migration Service)**

**DynamoDB = NoSql Database Service[Non Relational]**

For storing huge data we use **Datawarehouse**

For that we use **RedShift**


**ElastiCache** = **In memory database caching service**

ex: data in database which access frequently by the client can be stored in the cached memory. **ElastiCache** service has only 2 engines 

- **Redis**
- **Memcached**

in this service we can have low latency and high performance
